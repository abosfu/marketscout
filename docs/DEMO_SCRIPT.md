# MarketScout CLI Demo Script

Use this for a 45-second pitch, a 2-minute CLI walkthrough, and key talking points.

---

## 45-second pitch

> “MarketScout is a zero-friction strategy engine as a CLI. You pass industry, objective, city, and location; it fetches live headlines and job signals (no API keys), runs them through a strategy engine, and writes strategy.json, report.md, and report.html. Pain score, evidence-based problems, AI match categories, 30/60/90 plan, and ROI assumptions—all from real signals. It uses disk cache when live fetch fails so you can still get results from a previous run. Everything is schema-validated JSON.”

---

## 2-minute CLI walkthrough

1. **Install and run**  
   `pip install -r requirements.txt` then:
   ```bash
   python -m marketscout run --industry Construction --objective "Market entry" --city Vancouver --location "Vancouver, BC"
   ```

2. **What it does**  
   “The run command fetches live headlines (Google News RSS) and jobs (RSS) for the city and industry. No sample data at runtime. If the network fails but we have valid cached data in `.cache/marketscout/`, it uses that and warns. Otherwise it exits with a clear error.”

3. **Outputs**  
   “By default it writes to `out/`: strategy.json (full schema), report.md, and report.html. You can set `-o /path/to/dir`.”

4. **Terminal summary**  
   “Rich prints a header, signals used, score breakdown table, opportunity map table, and the output paths.”

5. **Modes**  
   “Strategy is generated by mock logic by default. Set `MARKETSCOUT_MODE=auto` and `OPENAI_API_KEY` to try OpenAI with fallback to mock.”

6. **Offline / pre-fetched data**  
   “Use `scout -o data/signals.json --include-jobs` to save signals, then `generate -i data/signals.json -o out/strategy.json` to generate from that file without network.”

---

## Key talking points

- **CLI-first**: No UI; primary interface is `marketscout run`. Real signals only at runtime; sample data is for tests/fixtures only.
- **Live signals**: Headlines from Google News RSS, jobs from job-related RSS. Retry with backoff; clear error if fetch fails (and no valid cache).
- **Caching**: `.cache/marketscout/` keyed by (city, industry, date). TTL configurable via `MARKETSCOUT_DISK_CACHE_TTL`. On fetch failure, valid cache is used and user is warned.
- **Multi-signal scoring**: Pain score combines headline and job signals; schema includes signals_used and score_breakdown.
- **Industry templates**: Objectives and bottlenecks per industry for consistent mock (and LLM) output.
- **Artifacts**: strategy.json, report.md, report.html from a single run.
